# Security policy

## Reporting a vulnerability

Do not open a public issue for an active or suspected vulnerability.

Use the repository's private **Security → Report a vulnerability** channel when it is available. Until a dedicated security address exists, contact the repository owner through an appropriate private GitHub channel.

Include:

- the affected route, component, or integration;
- clear reproduction steps or a minimal proof of concept;
- the expected and observed security impact;
- affected versions or commits;
- suggested mitigations, if known.

Do not include real credentials, payment information, access tokens, or personal data. Allow the maintainer reasonable time to investigate and coordinate a fix before public disclosure.
